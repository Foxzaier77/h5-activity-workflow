# v1.1

完成一次活动工作流实战迭代，主要更新并完善竞品分析规则问题。

## 更新内容

- 完善竞品活动分析规则，要求按单个活动输出，不打乱用户原有分类。
- 固化拼接缩略图方案，用单个活动总览缩略图提升文档直观性，避免截图过多导致文档冗长。
- 新增竞品活动逻辑拆解要求，覆盖活动目标、用户路径、核心机制、规则反推、激励反馈、可借鉴点和风险。
- 强化机制分析深度：说明为什么这么设计、希望用户做什么、用户为什么愿意做、平台希望达到什么效果，以及哪些交互或规则不适合直接照搬。
- 补充 Notion 文档整理规范，避免重复模块、空截图证据区和无法渲染的本地图片。
- 增加长任务复查和额度恢复后续跑机制，用于多文档竞品分析任务。
- 修复 agent 展示配置中的中文乱码。
